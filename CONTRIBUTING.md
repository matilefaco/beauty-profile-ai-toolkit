# Contributing

Thank you for helping improve Beauty Profile AI Toolkit. This project is intended to stay small, public, beginner-friendly, and safe for open-source collaboration.

## Safe contribution rules

Please do **not** contribute:

- Real API keys, tokens, service account files, credentials, or secrets.
- Real customer data, private profile data, phone numbers, emails, addresses, or screenshots.
- Private Nera SaaS source code, production prompts, product strategy, booking logic, payment logic, analytics logic, or Firebase production configuration.
- Claims that a generated bio can guarantee bookings, income, rankings, medical results, or client outcomes.
- Content copied from private businesses, competitors, clients, or paid prompt libraries without permission.

All examples should be fictional, minimal, and safe to publish.

## Using AI coding agents safely

AI coding agents can help with documentation, fictional examples, tests, and prompt evaluation. When using them for this repository:

- Ask agents to work only with files, issues, and pull requests from this public repository.
- Do not provide private Nera SaaS code, production prompts, booking logic, payment logic, credentials, production configuration, or real customer data.
- Keep any sample profiles, names, locations, services, and business details fictional and safe to publish.
- Review generated changes before opening a pull request, especially prompt wording, safety claims, and example data.
- Keep AI-assisted contributions small enough for maintainers to review easily.

## Good first contributions

- Add fictional example profiles for additional beauty specialties.
- Improve prompt wording for clarity, specificity, and safety.
- Add Portuguese or English variants of prompt templates.
- Create evaluation checklists for generated bios and headlines.
- Add lightweight TypeScript tests or examples.
- Improve documentation for beginners.

## Prompt contribution guidelines

Prompts should encourage copy that is:

- Short and practical.
- Grounded only in user-provided information.
- Warm without sounding exaggerated.
- Free of unsupported claims or invented credentials.
- Respectful of different beauty specialties, clients, tones, and locations.
- Clear about what information is missing instead of fabricating details.

## Development setup

```bash
npm install
npm run typecheck
```

This repository intentionally avoids heavy application frameworks. Keep additions simple unless there is a clear open-source benefit.

## Pull request checklist

Before opening a pull request, please confirm:

- [ ] I did not include secrets, credentials, production config, or real user data.
- [ ] I did not include private Nera SaaS code, prompts, business logic, or strategy.
- [ ] Any examples I added are fictional.
- [ ] Prompt changes avoid unsupported claims and invented details.
- [ ] Documentation is clear for beginners.
- [ ] TypeScript changes pass `npm run typecheck` when applicable.
